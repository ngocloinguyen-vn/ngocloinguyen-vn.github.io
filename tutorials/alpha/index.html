Alpha Diversity in R: Getting Started

Date: July 01, 2025

1. What is Alpha Diversity?

Alpha diversity describes the biodiversity within a single sample. It is one of the most commonly used ecological metrics for characterizing microbial, plant, and animal communities.

Alpha diversity usually considers two major components:

  • Richness – the number of observed species (or ASVs/OTUs).
  • Evenness – how evenly individuals are distributed among species.

2. Species Accumulation and Rarefaction Curves

Species Accumulation Curve

A species accumulation curve displays the cumulative number of species detected as additional samples are collected. It helps evaluate whether the sampling effort is sufficient to capture community diversity.

Rarefaction Curve

Rarefaction curves estimate species richness after standardizing sequencing depth. This allows fair comparison among samples with different numbers of sequencing reads.

3. Common Alpha Diversity Indices

Index Description
Richness Counts the number of observed ASVs, OTUs, or species.
Shannon Considers both richness and evenness.
Simpson Gives greater weight to dominant species.

4. Should You Normalize the Data?

Richness

Presence/absence richness calculated using functions such as specnumber() does not technically require normalization because it only counts the number of taxa present.

When Normalization is Recommended

Normalization becomes important when:

  • Sequencing depth differs substantially among samples.
  • Comparing abundance-based diversity indices.
  • Performing statistical comparisons between groups.

Richness is generally more sensitive to sequencing depth than Shannon or Simpson indices because deeper sequencing detects more rare taxa.

5. Common Normalization Methods

Method Recommended Use
Rarefaction Standard approach for alpha diversity.
CSS Useful for compositional microbiome data.
TSS Simple total-sum scaling.
DESeq2 Mainly for differential abundance analysis.

In this tutorial, we use rarefaction before calculating Normalized Richness and Shannon Diversity to reduce bias caused by unequal sequencing depth.

6. Workflow for Alpha Diversity Analysis

A typical alpha diversity workflow consists of the following steps:

  1. Prepare the ASV abundance table.
  2. Filter low-quality samples or extremely rare taxa.
  3. Normalize sequencing depth (optional but recommended).
  4. Calculate diversity indices.
  5. Visualize results.
  6. Perform statistical comparisons among groups.

7. Preparing Your Data

The input community matrix should contain:

  • Rows = samples
  • Columns = ASVs or OTUs
  • Cell values = read counts

Example:


           ASV1 ASV2 ASV3 ASV4
Sample1      15    0   24    8
Sample2       2   35    1    0
Sample3      20    5    9   14

Metadata should be stored separately and contain information such as treatment, sampling site, depth, season, or environmental variables.
