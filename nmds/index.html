Starting with nMDS in R

Date: July 01, 2025

1. What is NMDS?

Non-metric Multidimensional Scaling (NMDS) is a powerful ordination technique used to reduce the dimensionality of complex ecological data, such as OTU or ASV tables, by representing patterns in two or three dimensions.

In NMDS plots:

  • Each point represents a sample.
  • The closer the two points are, the more similar their composition is based on the input variables (e.g., OTUs/ASVs).

Unlike PCA or PCoA, NMDS is non-parametric-it’s rank-based and does not assume normally distributed data, which is ideal for microbial ecology datasets that often contain many rare taxa (long-tailed distributions).

2. How NMDS Works

  • Non-metric: Based on ranks of dissimilarities, not the actual distances.
  • Iterative: NMDS runs multiple iterations to find a configuration that best preserves the rank order of pairwise dissimilarities.
  • Input: Requires a distance matrix (e.g., Bray-Curtis, Jaccard).

3. How to perform NMDS

To perform NMDS, we generally follow these steps using the vegan package.

  • Prepare your data
    • Your input should be a community matrix (samples as rows, ASV as columns).
    • Consider filtering rare taxa or low-read samples beforehand.

  • Convert to metagenomeSeq Object & CSS Normalize

    Here, the CSS method is used for normalization.
    There are a few alternative normalization methods commonly used for microbiome or ecological count data, each with its own strengths depending on your analysis goals:

    • Total Sum Scaling (TSS): Normalizes each sample by dividing by the total number of reads; simple but sensitive to library size differences and does not handle sparsity well.
    • Rarefaction: Random subsampling to a fixed depth; useful for equalizing library sizes but can discard valuable data and introduce randomness.
    • Relative Abundance: Handles compositionality by log-transforming the ratio of each count to the geometric mean; often used in conjunction with tools like ALDEx2 or compositions package.
    • Centered Log-Ratio (CLR) Transformation: Converts counts to proportions (e.g., counts per ASV divided by total sample count); good for compositional comparisons but not suitable for abundance-sensitive metrics.
    • Variance Stabilizing Transformation (VST) from DESeq2: Reduces heteroscedasticity in count data, suitable for downstream statistical analysis.
    • Quantile or Rank Normalization: Aligns distributions across samples, mostly used in gene expression but applicable to microbiome data for certain comparisons.

Each method has trade-offs, so your choice should align with your research question and downstream statistical requirements.


# Load required packages
library(metagenomeSeq)    # For CSS normalization

# Transpose the input community matrix so that samples are rows, OTUs/ASVs are columns
ASV_read_count = as.data.frame(t(com))

# Convert to metagenomeSeq object
metaSeqObject = newMRexperiment(ASV_read_count)

# Compute normalization factor using CSS method
metaSeqObject_CSS = cumNorm(metaSeqObject, p = cumNormStatFast(metaSeqObject))

# Extract CSS-normalized counts (log-transformed)
ASV_read_count_CSS = data.frame(MRcounts(metaSeqObject_CSS, norm = TRUE, log = TRUE))

# Transpose so that samples are rows and ASVs are columns (vegan expects this)
m_com = as.matrix(t(ASV_read_count_CSS))
  • Run NMDS Ordination
    • metaMDS() performs NMDS, an ordination technique for visualizing patterns in community composition.
    • autotransform = FALSE disables automatic transformation, appropriate when prior normalization (e.g., CSS with log transform log = TRUE) is already applied.
    • Distance method: "bray" (Bray-Curtis) is widely used for ecological count data due to its sensitivity to compositional differences. 

# Run NMDS Ordination

set.seed(123)  # For reproducibility

# Perform NMDS using Bray-Curtis distance
nmds = metaMDS(m_com, autotransform = FALSE, distance = "bray")

Quickly interpreting the output

  • NMDS stress: indicates the goodness of fit, with values < 0.1 considered good, < 0.2 acceptable, and > 0.2 suggesting a poor fit and reduced reliability.
  • NMDS1/NMDS2: Axes representing major gradients of variation in community composition.
  • Points: Represent samples; distances reflect dissimilarity.
  • Preparation for plotting:
    • Load required libraries: ggplot2, ggrepel, and dplyr.
    • Extract NMDS site scores and rename the axes to x and y.
    • Load metadata and merge relevant columns into the NMDS scores data frame.
    • Calculate convex hulls for each year_layer group based on NMDS coordinates.
    • Combine all hull points into a single data frame for plotting.
    • Define a color palette for the different layers (SUR, MID, BOT, SED).

    4. Interpretation of results

    • The NMDS plot shows visible clustering of samples according to sampling time (indicated by color) and possibly by sample type (indicated by shape).
    • These observed patterns suggest that temporal variation and sample type might influence community composition.
    • However, since this is a visual interpretation, no formal conclusion about group differences can be drawn yet.

    5. Suggestion for next steps

    • Perform an ANOSIM (Analysis of Similarities) test to statistically evaluate whether differences between groups (e.g., sampling times, sample types) are significant.
    • Use SIMPER (Similarity Percentages) analysis or Indicator Species Analysis to identify which species or ASVs contribute most to differences among groups.
    • Apply envfit to overlay environmental variables on the NMDS plot to understand how environmental factors relate to observed community patterns.
    • Consider exploring other ordination methods or multivariate statistics to further validate and interpret community structure (e.g., PCA, RDA).
